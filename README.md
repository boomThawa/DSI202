## บทนำ

รายงานฉบับนี้จัดทำขึ้นเพื่อสรุปรายละเอียดทั้งหมดของโครงการ MindVibe ซึ่งเป็นแพลตฟอร์มเช่าเสื้อผ้าออนไลน์ที่พัฒนาขึ้นโดยมีเป้าหมายหลักคือการส่งเสริมและขับเคลื่อนแฟชั่นที่ยั่งยืน รายงานจะเจาะลึกในแต่ละส่วนของโครงการ ตั้งแต่การวางแนวคิดเบื้องต้น การออกแบบประสบการณ์ผู้ใช้ (User Experience Design) ผ่าน User Stories, User Flow Diagrams, Sitemap, และ Paper Prototype ไปจนถึงขั้นตอนการติดตั้งระบบโดยใช้ Docker และการประเมินผลการทำงานของระบบตามเกณฑ์ที่กำหนดไว้ เป้าหมายสูงสุดของ MindVibe คือการทำให้ผู้ใช้สามารถเข้าถึงแฟชั่นที่หลากหลายได้อย่างสะดวกสบาย ควบคู่ไปกับการส่งเสริมการบริโภคอย่างมีความรับผิดชอบต่อสังคมและสิ่งแวดล้อม

## บทคัดย่อ

MindVibe ถือกำเนิดขึ้นจากความเชื่อที่ว่าอุตสาหกรรมแฟชั่นสามารถเติบโตไปพร้อมกับการสร้างความยั่งยืนได้ แพลตฟอร์มนี้ทำหน้าที่เป็นตัวกลางในการให้บริการเช่าเสื้อผ้าออนไลน์ เพื่อให้ผู้ใช้ได้สัมผัสประสบการณ์การแต่งกายที่หลากหลายสำหรับโอกาสต่างๆ ตั้งแต่ชุดราตรีสำหรับงานพิเศษไปจนถึงเสื้อผ้าลำลองสำหรับการใช้งานในชีวิตประจำวัน โดยไม่จำเป็นต้องลงทุนซื้อเสื้อผ้าใหม่ทั้งหมด แนวคิดนี้ไม่เพียงแต่ช่วยให้ผู้ใช้ประหยัดค่าใช้จ่าย แต่ยังเป็นส่วนสำคัญในการลดปริมาณขยะสิ่งทอที่เกิดจาก Fast Fashion และส่งเสริมการใช้ทรัพยากรอย่างคุ้มค่า

MindVibe มุ่งมั่นที่จะทำให้กระบวนการเช่าเสื้อผ้าเป็นเรื่องง่ายและสะดวกสบายที่สุดสำหรับผู้ใช้ ตั้งแต่การเลือกดูคอลเลคชั่นที่คัดสรรมาอย่างดี การระบุระยะเวลาเช่า การชำระเงินที่ปลอดภัย ไปจนถึงกระบวนการคืนสินค้าที่ไม่ยุ่งยาก ทั้งหมดนี้เพื่อให้ผู้ใช้สามารถสนุกกับการแต่งตัวได้อย่างเต็มที่ ในขณะเดียวกันก็เป็นส่วนหนึ่งของการขับเคลื่อนแนวคิดแฟชั่นหมุนเวียน (Circular Fashion) ในประเทศไทย

แพลตฟอร์มนี้ถูกพัฒนาขึ้นบนสแตกเทคโนโลยีที่ทันสมัยและมีประสิทธิภาพสูง คือ Django ซึ่งเป็น Web Framework ที่ได้รับความนิยมของ Python และ Docker ซึ่งเป็นเครื่องมือสำหรับการจัดการ Container เทคโนโลยีทั้งสองนี้ทำงานร่วมกันเพื่อรองรับการพัฒนาที่รวดเร็ว การปรับใช้ (Deployment) ที่มีความปลอดภัยและสม่ำเสมอในสภาพแวดล้อมต่างๆ รวมถึงความสามารถในการขยายระบบ (Scalability) เพื่อรองรับจำนวนผู้ใช้และปริมาณข้อมูลที่เพิ่มขึ้นในอนาคต

### ความน่าสนใจและประโยชน์ต่อชุมชนของ MindVibe

* **ส่งเสริมความยั่งยืนอย่างเป็นรูปธรรม:** MindVibe เป็นทางเลือกที่ชัดเจนในการต่อสู้กับผลกระทบด้านลบของ Fast Fashion เช่น การใช้ทรัพยากรจำนวนมากในการผลิต การสร้างมลพิษ และปัญหาขยะสิ่งทอ การสนับสนุนให้เกิดการใช้ซ้ำเสื้อผ้าเป็นการยืดอายุการใช้งานของผลิตภัณฑ์ ลดความต้องการในการผลิตใหม่ และลดปริมาณขยะที่ต้องกำจัด ซึ่งส่งผลดีต่อสิ่งแวดล้อมในระยะยาว

* **เพิ่มการเข้าถึงแฟชั่นสำหรับทุกคน:** แพลตฟอร์มนี้ทำให้ผู้คนสามารถเข้าถึงเสื้อผ้าจากแบรนด์ที่มีชื่อเสียงหรือสไตล์ที่อาจมีราคาสูงได้ในราคาที่ย่อมเยากว่าการซื้อขาด เปิดโอกาสให้ผู้ใช้ได้ทดลองสวมใส่เสื้อผ้าที่หลากหลายตามเทรนด์หรือความต้องการสำหรับโอกาสพิเศษต่างๆ โดยไม่ต้องแบกรับภาระทางการเงินที่สูง

* **สร้างประสบการณ์การแต่งตัวที่ยืดหยุ่นและไร้กังวล:** MindVibe มอบความสะดวกสบายในการเข้าถึงเสื้อผ้าที่หลากหลาย โดยไม่ต้องกังวลเรื่องพื้นที่จัดเก็บในตู้เสื้อผ้า หรือปัญหาเสื้อผ้าที่ซื้อมาแล้วไม่ได้ใช้งาน การเช่าเสื้อผ้าช่วยให้ผู้ใช้สามารถปรับเปลี่ยนสไตล์การแต่งตัวได้บ่อยครั้งตามต้องการ

* **เป็นส่วนหนึ่งของการสร้างเศรษฐกิจหมุนเวียน:** MindVibe ไม่ได้เป็นเพียงแค่ธุรกิจเช่าเสื้อผ้า แต่เป็นกลไกสำคัญในการขับเคลื่อนเศรษฐกิจหมุนเวียนในอุตสาหกรรมแฟชั่น โดยการทำให้เสื้อผ้ามีการหมุนเวียนใช้งานซ้ำๆ มากที่สุด ลดการพึ่งพาทรัพยากรใหม่ในการผลิต และสร้างมูลค่าจากผลิตภัณฑ์ที่มีอยู่แล้ว

## 2. การนำเสนอระบบที่สอดคล้องกับการออกแบบ

ระบบ MindVibe ได้รับการพัฒนาโดยยึดหลักการออกแบบที่เน้นผู้ใช้เป็นศูนย์กลาง และตรวจสอบให้แน่ใจว่าทุกองค์ประกอบของระบบสอดคล้องกับแผนที่วางไว้ตั้งแต่เริ่มต้น

### 2.1 User Stories และ User Flow Diagrams

User Stories เป็นเครื่องมือสำคัญในการทำความเข้าใจความต้องการของผู้ใช้และกำหนดขอบเขตการทำงานของระบบ MindVibe ได้กำหนด User Stories หลัก 4 เรื่อง ซึ่งครอบคลุมการใช้งานพื้นฐานและสำคัญที่สุดของแพลตฟอร์ม ดังนี้:

**User Story 1: การสมัครสมาชิกเพื่อเข้าใช้งานแอป**

* **เหตุผล:** การสมัครสมาชิกเป็นขั้นตอนแรกที่จำเป็นเพื่อให้ผู้ใช้สามารถเข้าถึงฟังก์ชันการทำงานที่ต้องมีการระบุตัวตน เช่น การเช่าสินค้า การจัดการตะกร้า หรือการดูประวัติ

* **คำอธิบาย:** ผู้ใช้ใหม่สามารถสร้างบัญชีได้โดยตรงภายในแอปพลิเคชัน โดยมีทางเลือกในการลงทะเบียนด้วยอีเมลและรหัสผ่านที่ตั้งขึ้นเอง หรือใช้บัญชีโซเชียลมีเดียที่มีอยู่แล้ว (เช่น Facebook หรือ Google) เพื่อความรวดเร็วและลดขั้นตอน

* **Acceptance Criteria:**

    * ผู้ใช้ต้องสามารถกรอกข้อมูลที่จำเป็นในแบบฟอร์มการสมัคร (เช่น อีเมล, รหัสผ่าน) หรือเลือกช่องทางการสมัครผ่านโซเชียลมีเดีย

    * หลังจากดำเนินการสมัครสมาชิกสำเร็จ ระบบจะต้องนำผู้ใช้ไปยังหน้าแรกของแอปพลิเคชัน (Dashboard) โดยอัตโนมัติ พร้อมแสดงข้อความต้อนรับหรือยืนยันการสมัคร

    * หากผู้ใช้กรอกข้อมูลไม่ถูกต้อง (เช่น รูปแบบอีเมลไม่ถูกต้อง, รหัสผ่านไม่ตรงตามข้อกำหนด) หรือเกิดข้อผิดพลาดในกระบวนการสมัคร ระบบจะต้องแสดงข้อความแจ้งข้อผิดพลาดที่ชัดเจนและเป็นมิตรต่อผู้ใช้ เพื่อให้ผู้ใช้สามารถแก้ไขได้

* **การทำงานของระบบ:** User Flow Diagram สำหรับ User Story นี้จะแสดงขั้นตอนอย่างละเอียด เริ่มตั้งแต่ผู้ใช้เข้าสู่หน้า Home (ซึ่งอาจเป็นหน้า Landing Page หรือหน้า Login/Sign Up หากยังไม่ได้เข้าสู่ระบบ) คลิกที่ปุ่ม "Sign Up" ระบบจะเปลี่ยนเส้นทางไปยังหน้าสมัครสมาชิก ผู้ใช้กรอกข้อมูลในฟอร์มหรือเลือกสมัครผ่านโซเชียลมีเดีย เมื่อกดปุ่ม "Create Account" ระบบ Backend จะทำการตรวจสอบและสร้างบัญชีผู้ใช้ในฐานข้อมูล หากสำเร็จจะส่งผู้ใช้ไปยังหน้า Dashboard หากล้มเหลวจะแสดงข้อความผิดพลาดบนหน้าสมัครสมาชิก

**User Story 2: ผู้ใช้ต้องการค้นหาสินค้าในแอปเพื่อหาสินค้าที่ต้องการซื้อ/เช่า**

* **เหตุผล:** เพื่อให้ผู้ใช้สามารถค้นหาสินค้าที่ตรงกับความต้องการได้อย่างรวดเร็วและมีประสิทธิภาพ ไม่ว่าผู้ใช้จะทราบชื่อสินค้าที่ต้องการ หรือต้องการเรียกดูสินค้าตามประเภทและหมวดหมู่ต่างๆ

* **คำอธิบาย:** ผู้ใช้สามารถใช้แถบค้นหา (Search Bar) ซึ่งมักจะอยู่ที่ส่วนบนของหน้าจอ เพื่อกรอกคำค้นหาที่เกี่ยวข้องกับสินค้า (เช่น ชื่อสินค้า, สี, สไตล์) หรือเลือกเรียกดูสินค้าตามหมวดหมู่ที่ระบบจัดเตรียมไว้ (เช่น ชุดเดรส, เสื้อเชิ้ต, ชุดไปงานแต่ง)

* **Acceptance Criteria:**

    * ผู้ใช้ต้องสามารถกรอกคำค้นหาในแถบค้นหา หรือคลิกเลือกหมวดหมู่สินค้าที่ต้องการเรียกดูได้

    * ระบบต้องแสดงผลลัพธ์การค้นหาเป็นรายการสินค้าที่มีความเกี่ยวข้องและตรงกับคำค้นหาหรืออยู่ในหมวดหมู่ที่ผู้ใช้เลือก

* **การทำงานของระบบ:** User Flow Diagram จะแสดงเส้นทางของผู้ใช้จากหน้า Dashboard ไปยังแถบค้นหาหรือส่วนแสดงหมวดหมู่สินค้า เมื่อผู้ใช้กรอกคำค้นหาและกดค้นหา หรือคลิกที่หมวดหมู่ ระบบจะประมวลผลและดึงข้อมูลสินค้าที่เกี่ยวข้องกลับมาแสดงผลบนหน้า Search Results Page ผู้ใช้สามารถเลื่อนดูรายการสินค้าและคลิกที่สินค้าแต่ละรายการเพื่อดูรายละเอียดเพิ่มเติม ซึ่งจะนำไปยังหน้า Product Details Page

**User Story 3: ผู้ใช้ต้องการเพิ่มสินค้าลงในตะกร้าและทำการชำระเงิน**

* **เหตุผล:** นี่คือขั้นตอนสำคัญในกระบวนการแปลงผู้เยี่ยมชมให้เป็นผู้เช่า การที่ผู้ใช้สามารถเพิ่มสินค้าที่สนใจลงในตะกร้าและดำเนินการชำระเงินได้อย่างราบรื่นเป็นสิ่งจำเป็นสำหรับความสำเร็จของแพลตฟอร์ม

* **คำอธิบาย:** หลังจากที่ผู้ใช้ได้ตรวจสอบรายละเอียดสินค้าที่หน้า Product Details และตัดสินใจที่จะเช่าสินค้านั้น ผู้ใช้สามารถระบุระยะเวลาที่ต้องการเช่า (หากมีตัวเลือก) แล้วเพิ่มสินค้านั้นลงในตะกร้าสินค้า จากนั้นผู้ใช้สามารถไปยังหน้าตะกร้าสินค้าเพื่อตรวจสอบรายการทั้งหมด และดำเนินการเข้าสู่ขั้นตอนการชำระเงิน

* **Acceptance Criteria:**

    * ผู้ใช้ต้องสามารถเพิ่มสินค้าที่เลือก (โดยอาจมีการเลือกระยะเวลาเช่า) ลงในตะกร้าสินค้าได้สำเร็จ

    * ผู้ใช้ต้องสามารถไปยังหน้าตะกร้าสินค้าเพื่อตรวจสอบรายการสินค้าที่เลือกทั้งหมดได้

    * ผู้ใช้ต้องสามารถดำเนินการต่อจากหน้าตะกร้าสินค้าไปยังหน้าชำระเงิน (Checkout Page) และกรอกข้อมูลที่จำเป็น (เช่น ที่อยู่จัดส่ง, วิธีชำระเงิน) เพื่อทำการเช่าสินค้าให้เสร็จสมบูรณ์

* **การทำงานของระบบ:** User Flow Diagram จะเริ่มต้นจากหน้า Product Details Page เมื่อผู้ใช้คลิก "Add to Cart" ระบบจะจัดการเพิ่มสินค้าลงใน Session หรือ Database ของผู้ใช้ และอาจแสดงข้อความยืนยันหรือนำไปยังหน้า Cart Page โดยตรง บนหน้า Cart ผู้ใช้สามารถเห็นรายการสินค้าทั้งหมดที่เลือก และคลิก "Proceed to Checkout" ระบบจะนำไปยังหน้า Checkout Page ซึ่งผู้ใช้จะต้องกรอกข้อมูลการจัดส่งและเลือกวิธีการชำระเงิน เมื่อผู้ใช้กรอกรายละเอียดการชำระเงินและกดยืนยัน ระบบ Backend จะดำเนินการประมวลผลการชำระเงิน (ซึ่งอาจเป็นการจำลองในขั้นตอนนี้) และเมื่อสำเร็จจะนำผู้ใช้ไปยังหน้า Order Confirmation / Payment Success Page

**User Story 4: ผู้ใช้ต้องการตรวจสอบประวัติการเช่า-คืน**

* **เหตุผล:** เพื่อให้ผู้ใช้สามารถติดตามสถานะของคำสั่งเช่าที่ผ่านมาทั้งหมด รวมถึงสถานะของการคืนสินค้า ซึ่งเป็นสิ่งสำคัญสำหรับการจัดการบัญชีผู้ใช้และความโปร่งใสของบริการ

* **คำอธิบาย:** ผู้ใช้สามารถเข้าถึงส่วนของบัญชีผู้ใช้เพื่อดูรายการสินค้าทั้งหมดที่เคยเช่าในอดีต ระบบจะแสดงข้อมูลสำคัญเกี่ยวกับแต่ละรายการ เช่น ชื่อสินค้า วันที่เช่า ระยะเวลาเช่า และสถานะปัจจุบัน (เช่น กำลังเช่า, คืนแล้ว, รอการคืน) ผู้ใช้ยังสามารถดำเนินการแจ้งคืนสินค้าสำหรับรายการที่เช่าอยู่ได้จากหน้านี้

* **Acceptance Criteria:**

    * ระบบต้องแสดงรายการสินค้าทั้งหมดที่ผู้ใช้เคยเช่าในอดีตในรูปแบบที่เข้าใจง่าย พร้อมแสดงสถานะปัจจุบันของแต่ละรายการ

    * ผู้ใช้ต้องสามารถดำเนินการแจ้งคืนสินค้าสำหรับรายการที่เช่าอยู่ได้จากหน้านี้

    * เมื่อผู้ใช้แจ้งคืนสินค้า ระบบต้องอัปเดตสถานะของสินค้ารายการนั้นในหน้าประวัติการเช่าเป็น "Returned" หรือสถานะอื่นที่บ่งชี้ถึงกระบวนการคืนสินค้าที่กำลังดำเนินการ

* **การทำงานของระบบ:** User Flow Diagram จะแสดงเส้นทางจากหน้า Dashboard หรือเมนูหลักไปยังหน้า "History" หรือ "ประวัติการเช่า" ระบบจะดึงข้อมูลประวัติการเช่าของผู้ใช้จากฐานข้อมูลมาแสดงเป็นรายการ เมื่อผู้ใช้คลิกปุ่ม "Return Item" สำหรับรายการที่ต้องการคืน ระบบ Backend จะรับคำขอและอัปเดตสถานะของรายการนั้นในฐานข้อมูลเป็นสถานะที่เหมาะสม และอาจมีขั้นตอนเพิ่มเติมสำหรับการจัดการการคืนสินค้าทางกายภาพ

### 2.2 Sitemap

Sitemap ของแอปพลิเคชัน MindVibe ถูกออกแบบมาเพื่อให้ผู้ใช้สามารถเข้าถึงส่วนต่างๆ ของแพลตฟอร์มได้อย่างเป็นระบบและมีเหตุผล โครงสร้างหลักประกอบด้วยหน้าต่างๆ ดังนี้:

* **Home Page (หน้าแรก/Dashboard หลัง Login):** หน้าหลักหลังจากผู้ใช้เข้าสู่ระบบ มักจะแสดงสินค้าแนะนำ หมวดหมู่สินค้า หรือข้อมูลสรุปสำหรับผู้ใช้

* **Sign Up / Login Page:** หน้าสำหรับผู้ใช้ใหม่ที่ต้องการสมัครสมาชิก หรือผู้ใช้เดิมที่ต้องการเข้าสู่ระบบ

* **Search Results Page:** หน้าที่แสดงผลลัพธ์จากการค้นหาสินค้าด้วยคำค้นหา หรือจากการเลือกเรียกดูตามหมวดหมู่

* **Product Details Page:** หน้ารายละเอียดเฉพาะของสินค้าแต่ละรายการ แสดงข้อมูลครบถ้วน เช่น รูปภาพ, ราคาเช่า, ขนาด, สี, รายละเอียด, และตัวเลือกในการเลือกระยะเวลาเช่าและเพิ่มลงตะกร้า

* **Cart Page:** หน้าที่แสดงรายการสินค้าทั้งหมดที่ผู้ใช้ได้เพิ่มลงในตะกร้า สามารถตรวจสอบจำนวน ราคารวม และอาจสามารถแก้ไขหรือลบรายการสินค้าได้

* **Checkout Page:** หน้าสำหรับดำเนินการชำระเงิน ผู้ใช้กรอกข้อมูลที่อยู่จัดส่ง เลือกวิธีการชำระเงิน และกรอกรายละเอียดการชำระเงิน

* **Order Confirmation Page (Payment Success):** หน้าที่แสดงเมื่อกระบวนการสั่งซื้อและชำระเงินเสร็จสมบูรณ์ แสดงรายละเอียดการสั่งซื้อและหมายเลขออเดอร์

* **History Page (ประวัติการเช่า-คืน):** หน้าที่แสดงรายการสินค้าทั้งหมดที่ผู้ใช้เคยเช่าในอดีต พร้อมสถานะปัจจุบันของแต่ละรายการ และตัวเลือกในการแจ้งคืนสินค้า

การนำทางระหว่างหน้าเหล่านี้ถูกออกแบบให้มีความราบรื่นและสอดคล้องกับ User Flow Diagrams เพื่อมอบประสบการณ์ผู้ใช้ที่ดี

### 2.3 Paper Prototype และการวัดเวลา

การทดสอบ Paper Prototype เป็นขั้นตอนสำคัญในการตรวจสอบการออกแบบ UI/UX ในช่วงต้นของการพัฒนา แม้ว่ารายงานจะไม่ได้ระบุจำนวนผู้เข้าร่วมทดสอบที่แน่นอน แต่ได้กล่าวถึงผลลัพธ์ที่น่าพอใจจากการทดสอบนี้ พบว่าผู้ใช้ส่วนใหญ่สามารถทำตาม User Stories หลักที่กำหนดไว้ได้ภายในระยะเวลาที่ค่อนข้างสั้น โดยเฉลี่ยไม่เกิน 30 วินาทีต่อ User Story

ผลการทดสอบนี้มีความสำคัญอย่างยิ่ง เนื่องจากชี้ให้เห็นว่าการออกแบบ UI เบื้องต้นที่ร่างไว้บนกระดาษมีความเข้าใจง่าย (Intuitive) และมีประสิทธิภาพในการใช้งานจริง โดยเฉพาะอย่างยิ่งในกระบวนการที่สำคัญ เช่น การเลือกชุดและการชำระเงินที่ใช้เวลาน้อย ซึ่งสะท้อนถึงการออกแบบที่เน้นความรวดเร็วและความสะดวกในการทำธุรกรรม การทดสอบนี้ช่วยยืนยันว่าแนวคิดการออกแบบพื้นฐานมีความแข็งแกร่งก่อนที่จะนำไปพัฒนาเป็นระบบจริง

### 2.4 Color Theme + Font

การเลือกใช้สีและฟอนต์ในแอปพลิเคชัน MindVibe ไม่ได้เป็นเพียงการเลือกเพื่อความสวยงาม แต่เป็นการสร้างอัตลักษณ์ของแบรนด์ที่ชัดเจนและสื่อสารคุณค่าหลักของโครงการ:

* **สีหลัก: สีเขียว (#00A300)**

    * การใช้สีเขียวเป็นสีหลัก โดยเฉพาะในปุ่ม Call-to-action ที่สำคัญ ("Proceed to Checkout", "Confirm Payment") เป็นการสื่อสารโดยตรงถึงแนวคิดความยั่งยืน (Sustainability) และความเป็นมิตรต่อสิ่งแวดล้อม (Environmental Friendliness) ซึ่งเป็นหัวใจสำคัญของ MindVibe สีเขียวยังสามารถสื่อถึงการเติบโต ความสดชื่น และความเชื่อมั่นได้อีกด้วย

* **สีรอง: สีชมพู (#FF3377)**

    * สีชมพูถูกใช้เป็นสีรองในส่วนประกอบต่างๆ เช่น โลโก้ "MindVibe" การใช้สีชมพูนี้ช่วยเพิ่มความรู้สึกสดใส มีชีวิตชีวา และดึงดูดสายตา ซึ่งเชื่อมโยงกับโลกของแฟชั่นและความสนุกสนานในการแต่งตัว เป็นการผสมผสานระหว่างความยั่งยืนกับความน่าสนใจด้านแฟชั่น

* **พื้นหลัง: สีขาวและดำเป็นหลัก (หรือสีเทาอ่อน)**

    * การใช้สีพื้นหลังที่เป็นกลาง เช่น สีขาว สีดำ หรือสีเทาอ่อน ช่วยสร้างความรู้สึกสะอาดตา เรียบง่าย และทันสมัย สีพื้นหลังเหล่านี้มีบทบาทสำคัญในการขับเน้นเนื้อหาและรูปภาพสินค้าให้โดดเด่นและน่าสนใจ ทำให้ผู้ใช้สามารถโฟกัสที่สินค้าได้ง่าย

* **ฟอนต์: ตระกูล Sans-serif (เช่น Roboto หรือ Prompt)**

    * การเลือกใช้ฟอนต์ตระกูล Sans-serif เช่น Roboto หรือ Prompt เป็นการตัดสินใจที่เน้นด้านการใช้งาน (Usability) ฟอนต์ประเภทนี้มักจะอ่านง่ายบนหน้าจออุปกรณ์ดิจิทัลต่างๆ ทั้งบน Desktop และ Mobile ทำให้ข้อความมีความชัดเจนและเข้าถึงได้ดี การใช้ฟอนต์ Sans-serif ยังช่วยสร้างความรู้สึกทันสมัยและเป็นมิตร

การผสมผสานสีและฟอนต์เหล่านี้ช่วยเสริมสร้างภาพลักษณ์โดยรวมของ MindVibe ในฐานะแพลตฟอร์มที่น่าเชื่อถือ ทันสมัย ใส่ใจสิ่งแวดล้อม และมอบประสบการณ์การใช้งานที่น่าพึงพอใจให้กับผู้ใช้

## 3. ขั้นตอนการใช้งานตาม User Stories

รายงานได้อธิบายขั้นตอนการใช้งานสำหรับแต่ละ User Story ไว้อย่างละเอียด ซึ่งสะท้อนถึง User Flow Diagrams ที่ออกแบบไว้:

**User Story 1: การสมัครสมาชิกเพื่อเข้าใช้งานแอป**

1.  ผู้ใช้เข้าสู่หน้า Home Page ของ MindVibe (หากยังไม่ได้เข้าสู่ระบบ อาจเป็นหน้า Landing Page หรือหน้า Login/Sign Up โดยตรง)

2.  ผู้ใช้คลิกที่ปุ่ม "Sign Up" หรือ "ลงทะเบียน" ที่เห็นบนหน้าจอ

3.  ระบบนำทางผู้ใช้ไปยังหน้า Sign Up / Login Page

4.  ผู้ใช้เลือกวิธีการสมัคร: กรอกอีเมลและตั้งรหัสผ่านในฟอร์มที่กำหนด หรือคลิกปุ่มสำหรับสมัครผ่านบัญชีโซเชียลมีเดีย (Facebook/Google)

5.  ผู้ใช้กดปุ่ม "Create Account" หรือปุ่มยืนยันการสมัคร

6.  **กรณีสำเร็จ:** ระบบตรวจสอบข้อมูลและสร้างบัญชีใหม่ในฐานข้อมูล จากนั้นนำผู้ใช้ไปยังหน้า Home/Dashboard พร้อมแสดงข้อความยืนยันการสมัครสำเร็จ (เช่น "ยินดีต้อนรับสู่ MindVibe!")

7.  **กรณีผิดพลาด:** ระบบตรวจสอบข้อมูลและพบข้อผิดพลาด (เช่น อีเมลซ้ำ, รหัสผ่านไม่ตรงตามข้อกำหนด, ข้อมูลไม่ครบถ้วน) ระบบจะแสดงข้อความแจ้งข้อผิดพลาดที่เฉพาะเจาะจงบนหน้า Sign Up / Login Page เพื่อให้ผู้ใช้แก้ไข

**User Story 2: ผู้ใช้ต้องการค้นหาสินค้าในแอป**

1.  ผู้ใช้เริ่มต้นที่หน้า Home/Dashboard หลังจากเข้าสู่ระบบ

2.  ผู้ใช้สังเกตเห็นแถบค้นหา (มักจะอยู่ที่ส่วนบนของหน้าจอ) หรือส่วนที่แสดงหมวดหมู่สินค้าต่างๆ

3.  **กรณีใช้แถบค้นหา:** ผู้ใช้คลิกที่แถบค้นหา พิมพ์คำค้นหาที่ต้องการ (เช่น "ชุดเดรสสีแดง", "เสื้อเชิ้ตผู้ชาย", "กางเกงยีนส์วินเทจ") และกดปุ่ม "ค้นหา" หรือ Enter

4.  **กรณีเลือกหมวดหมู่:** ผู้ใช้เลื่อนดูหมวดหมู่สินค้าที่แสดงบนหน้า Home หรือในเมนู และคลิกที่หมวดหมู่ที่สนใจ (เช่น "ชุดไปงาน", "เสื้อผ้าทำงาน", "เสื้อผ้าลำลอง")

5.  ระบบจะประมวลผลคำขอและแสดงผลลัพธ์เป็นรายการสินค้าที่เกี่ยวข้องบนหน้า Search Results Page

6.  ผู้ใช้สามารถเลื่อนดูรายการสินค้าในหน้าผลลัพธ์ และคลิกที่รูปภาพหรือชื่อสินค้าที่สนใจเพื่อดูรายละเอียดเพิ่มเติม ระบบจะนำไปยังหน้า Product Details Page สำหรับสินค้านั้น

**User Story 3: ผู้ใช้ต้องการเพิ่มสินค้าลงในตะกร้าและทำการชำระเงิน**

1.  ผู้ใช้เข้าชมหน้า Product Details Page ของสินค้าที่ต้องการเช่า

2.  ผู้ใช้ตรวจสอบรายละเอียดของสินค้าอย่างละเอียด เช่น รูปภาพสินค้าหลายมุมมอง, ราคาเช่าต่อระยะเวลา, ขนาดที่มีให้เลือก, สี, วัสดุ, และคำอธิบายเพิ่มเติม

## 4. การติดตั้งและใช้งาน (อ้างอิง Docker)

การติดตั้งและใช้งานระบบ MindVibe ถูกออกแบบให้มีความสะดวกและสม่ำเสมอในสภาพแวดล้อมที่แตกต่างกันโดยใช้ Docker เป็นหลัก:

### 4.1 ข้อกำหนดเบื้องต้น:

เพื่อให้สามารถติดตั้งและรันระบบ MindVibe ได้ ผู้ใช้จำเป็นต้องมีโปรแกรมพื้นฐานสองอย่างติดตั้งอยู่บนเครื่องคอมพิวเตอร์:

* **Docker:** แพลตฟอร์มสำหรับการพัฒนา จัดส่ง และรันแอปพลิเคชันโดยใช้ Container

* **Docker Compose:** เครื่องมือสำหรับกำหนดและจัดการแอปพลิเคชันแบบ Multi-container Docker

ต้องแน่ใจว่าทั้งสองโปรแกรมนี้ได้รับการติดตั้งและทำงานได้อย่างถูกต้องบนระบบปฏิบัติการที่ใช้งาน

### 4.2 ขั้นตอนการติดตั้ง:

ขั้นตอนการติดตั้งระบบ MindVibe มีดังนี้:

1.  **ดาวน์โหลดหรือโคลน Repository ของโครงการ:** เปิดโปรแกรม Terminal (บน macOS/Linux) หรือ Command Prompt / PowerShell (บน Windows) จากนั้นใช้คำสั่ง `git clone` เพื่อดาวน์โหลดซอร์สโค้ดทั้งหมดของโครงการ MindVibe จาก GitHub Repository มายังเครื่องคอมพิวเตอร์ของคุณ:

    ```bash
    git clone [https://github.com/yourusername/dsi205.git](https://github.com/yourusername/dsi205.git)
    ```

    (ข้อควรจำ: ต้องแทนที่ `yourusername` ด้วยชื่อผู้ใช้ GitHub จริงของผู้จัดทำโครงการ)

2.  **เข้าสู่ Directory ของโครงการ:** ใช้คำสั่ง `cd` (Change Directory) เพื่อย้ายตำแหน่งปัจจุบันใน Terminal หรือ Command Prompt ไปยัง Directory ของ Repository ที่เพิ่งโคลนมา:

    ```bash
    cd dsi205
    ```

    ภายใน Directory นี้จะมีไฟล์ `docker-compose.yml` ซึ่งเป็นไฟล์กำหนดค่าสำหรับ Docker Compose อยู่

3.  **สร้างและเริ่มต้น Container:** ใช้คำสั่ง `docker-compose up --build` เพื่อให้ Docker Compose อ่านไฟล์ `docker-compose.yml` และดำเนินการตามที่กำหนดไว้ คำสั่งนี้จะทำการ:

    * Build Docker Image ที่จำเป็นสำหรับแต่ละบริการ (Service) ที่กำหนดในไฟล์ `docker-compose.yml` (เช่น Image สำหรับ Web Application ที่ใช้ Django, Image สำหรับ Database Server ที่ใช้ PostgreSQL) หาก Image ยังไม่มีอยู่ หรือมีการเปลี่ยนแปลงใน Dockerfile

    * สร้าง Container จาก Image ที่ Build หรือที่มีอยู่แล้ว

    * เชื่อมโยง Container เข้าด้วยกันตามที่กำหนด (เช่น Container ของ Django เชื่อมต่อไปยัง Container ของ Database)

    * เริ่มต้นการทำงานของ Container ทั้งหมด

    ```bash
    docker-compose up --build
    ```

    หากต้องการให้ Container ทำงานในพื้นหลังโดยไม่แสดง Log ใน Terminal และสามารถปิด Terminal ได้ ให้เพิ่ม Option `-d` (detached mode):

    ```bash
    docker-compose up --build -d
    ```

    กระบวนการนี้อาจใช้เวลาสักครู่ในการดาวน์โหลด Base Images, Build Images, และเริ่มต้น Container ต่างๆ

### 4.3 การเข้าใช้งาน:

เมื่อคำสั่ง `docker-compose up --build` (หรือ `-d`) ทำงานเสร็จสิ้น และ Container ทั้งหมดอยู่ในสถานะ Running แสดงว่าระบบ MindVibe พร้อมใช้งานแล้ว คุณสามารถเข้าใช้งานแอปพลิเคชันผ่านเว็บเบราว์เซอร์ใดก็ได้ โดยพิมพ์ URL ต่อไปนี้ในแถบที่อยู่:


http://localhost:8000/


นี่คือ URL ที่ถูกกำหนดให้ Web Server (Django Development Server ที่รันภายใน Container) รับฟังการเชื่อมต่อจากภายนอก

โครงการ MindVibe ได้รับการพัฒนาและนำเสนออย่างมีคุณภาพสูงตามแผนที่วางไว้ทุกประการ ระบบที่สร้างขึ้นสามารถทำงานได้จริงตาม User Stories และการออกแบบที่ได้กำหนดไว้ตั้งแต่เริ่มต้น การเลือกใช้ Docker เป็นเทคโนโลยีหลักในการจัดการสภาพแวดล้อมและการปรับใช้ (Deployment) เป็นการตัดสินใจที่ชาญฉลาด ซึ่งช่วยให้กระบวนการติดตั้งมีความสะดวก รวดเร็ว และมีความสม่ำเสมอในสภาพแวดล้อมที่แตกต่างกัน ทำให้โครงการนี้มีความพร้อมสำหรับการพัฒนาต่อยอดและการนำไปใช้งานจริงในอนาคต

MindVibe ไม่ได้เป็นเพียงแพลตฟอร์มเช่าเสื้อผ้าออนไลน์ที่มีฟังก์ชันการทำงานครบถ้วนตามที่ออกแบบไว้เท่านั้น แต่ยังมีความสำคัญอย่างยิ่งในด้านการส่งเสริมแนวคิดแฟชั่นที่ยั่งยืน ซึ่งเป็นประเด็นที่มีความสำคัญมากขึ้นในปัจจุบัน การมีแพลตฟอร์มเช่น MindVibe เป็นการนำเสนอทางเลือกในการบริโภคเสื้อผ้าที่มีความรับผิดชอบต่อสิ่งแวดล้อมและสังคม ซึ่งเป็นประโยชน์อย่างยิ่งต่อทั้งผู้ใช้ที่ต้องการทางเลือกใหม่ๆ และต่อชุมชนโดยรวมในการลดผลกระทบด้านลบจากอุตสาหกรรมแฟชั่น

นอกจากนี้ การจัดการ GitHub Repository ที่เป็นระบบ การ Commit งานอย่างสม่ำเสมอและมีคุณภาพ พร้อมด้วยไฟล์ README.md ที่ให้ข้อมูลอย่างครบถ้วนและชัดเจน แสดงให้เห็นถึงแนวทางการพัฒนาซอฟต์แวร์ที่ดี ซึ่งเอื้อต่อการติดตามความคืบหน้า การบำรุงรักษาโค้ด และการทำงานร่วมกันในอนาคต (หากมีการขยายทีมพัฒนา)

โดยสรุป โครงการ MindVibe ได้บรรลุวัตถุประสงค์ที่ตั้งไว้ทั้งหมด แสดงให้เห็นถึงความสามารถในการวิเคราะห์ ออกแบบ พัฒนา และนำเสนอโซลูชันซอฟต์แวร์ที่ตอบสนองความต้องการทางธุรกิจและสังคมได้อย่างมีประสิทธิภาพ พร้อมทั้งใช้เครื่องมือและกระบวนการพัฒนาที่เป็นมาตรฐานและทันสมัย โครงการนี้มีศักยภาพสูงในการพัฒนาต่อยอดเพื่อเป็นแพลตฟอร์มเช่าเสื้อผ้าออนไลน์ชั้นนำที่ขับเคลื่อนแฟชั่นที่ยั่งยืน.
